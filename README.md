# ✋ On-Device Real-Time Custom Hand Gesture Recognition
> **본 프로젝트는 기존의 정해진 손동작만 인식하는 것 외에, 사용자가 원하는 커스텀 제스처를 직접 등록하고 스마트폰 등의 기기에서 실시간으로 구동하는 것을 목표로 합니다.**

---

## 🛠️ 기술 스택 (Tech Stack)
* **Framework**: Google MediaPipe (Hands & Tasks API)
* **Environment**: Google Colab (Python)
* **Development**: Android Studio (Java)
* **Model Format**: TensorFlow Lite (`.task`)

---

## 🚀 How to Use

### 📂 1단계: 나만의 제스처 사진 모으기 (Data Collection)
* **목적**: 인공지능이 손 모양을 배울 수 있도록 기준이 되는 사진을 준비합니다.
* **방법**: 
  * 인식시키고 싶은 손 모양(예: 하트, V, 주먹 등)을 정합니다.
  * 각 제스처당 **최소 50장 이상** 다양한 각도와 배경에서 사진을 촬영합니다.
  * 사진 수가 너무 적으면 컴퓨터가 손 모양을 잘 구별하지 못하므로 넉넉하게 준비하는 것이 좋습니다.

### 💻 2단계: Google Colab에서 인공지능 학습시키기 (Model Training)
* **목적**: 준비한 사진들을 학습시켜서 스마트폰용 인공지능 모델 파일(`.task`)을 만듭니다.
* **방법**:
  1. 구글 코랩(Google Colab) 환경을 열고 `mediapipe-model-maker` 라이브러리를 설치합니다.
  2. 수집한 사진을 업로드한 뒤, 이미지 속 21개의 손가락 관절 좌표(Landmarks)를 추출하는 코드를 실행합니다.
  3. 추출된 데이터를 바탕으로 커스텀 분류기(Classifier) 학습을 진행합니다.
  4. 학습이 완료되면 기기에서 즉시 읽을 수 있는 **`gesture_recognizer.task`** 파일로 다운로드합니다.

### 📱 3단계: 기기에 넣고 실시간 인식하기 (On-Device Inference)
* **목적**: 만든 모델을 프로그램에 이식하여 카메라 화면으로 손동작을 실시간 알아맞힙니다.
* **방법**:
  1. 다운로드한 `.task` 모델 파일을 안드로이드 앱 등의 자원(Assets) 폴더에 배치합니다.
  2. 카메라에서 들어오는 실시간 영상 프레임을 분석기(`ImageAnalysis`)로 계속 읽어옵니다.
  3. `MediaPipe Hands` API가 화면 속 손가락 관절의 위치를 실시간으로 추적합니다.
  4. 변환된 데이터가 내가 만든 모델 파일과 비교 분석되어, 현재 어떤 제스처(예: rock, paper 등)를 취하고 있는지 화면에 즉시 결과로 출력됩니다.

---

## ⚠️ 확인된 한계점 및 주의사항 (Limitations)
* **사진 장수 영향**: 학습용 사진 장수가 너무 적으면 엉뚱한 제스처로 잘못 인식할 확률이 높아지므로 데이터 확보가 기본이 되어야 합니다.
* **손 겹침 현상**: 두 손이 너무 가깝게 붙거나 겹치면 손가락 관절 좌표를 쫓아가는 정확도가 떨어질 수 있습니다.
* **정적 이미지 기반**: 본 시스템은 움직이는 동적 모션이 아닌, 멈춰 있는 정적인 손 모양 프레임을 기준으로 학습하고 판단합니다.
